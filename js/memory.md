## 引用计数算法(老东西)

当一个变量引用了一个对象或者这个对象的某一个属性时, 这个对象的引用计数增加. 

当这个变量不能再访问时, 对象的引用计数减少. 

当一个对象不能再被访问, 并且引用计数为0, 那么就可以回收了. 

但是存在循环引用问题

```javascript
function test() {
	var A = { a: undefined }
	var B = { b: undefined }
	A.a = B
	B.b = A
}
```

A.a引用了B, 因此B的引用计数+1. 

B.b引用了A, 因此A的引用计数+1, 同时因为A引用了B, 因此B的引用计数+1. 

最终导致两者的引用计数都不可能为0. 

这就出现了内存泄漏. 

## 分代式垃圾回收

v8将内存划分成两块, 一块较小, 一块较大. 

v8把对象分成了三代:
- 新生代
- 青年代
- 老年代

其中, 新生代和青年代处于较小的内存空间. 

### 新生代和青年代

较小的区域还会划分成两份相同的空间From和To. 

当有新对象时, 会放到From中. 

进行垃圾回收时, 遍历From中所有对象, 如果可达, 那么放到To中. 完成后, 清除From作为To, To变成From. 

同时, v8会记录偏差, 在一定范围内的就是从From活到To的, 也就是青年代. 

垃圾回收时, 如果可达, 并且是青年代(在范围内), 那么放到老年代中而不是To. 

### 老年代

老年代的内存空间扫描不频繁, 且时间用的久. 

老年代内存管理采用标记-清除和标记-整理. 

1. 遍历所有对象, 给活着的对象打上标记, 死了的对象"清除"
2. 在合适的时间点进行内存整理, 让内存变得紧密
